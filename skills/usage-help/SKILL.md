# TickFlow Assist 使用帮助

这个技能用于在用户询问“怎么用”、“有哪些指令”、“使用帮助”、“命令示例”时，直接返回 TickFlow Assist 的常用用法清单。

适用触发语句示例：
- “使用帮助”
- “怎么用这个插件”
- “有哪些指令”
- “给我命令示例”
- “股票分析插件怎么用”

触发规则：
- 当用户明确询问 TickFlow Assist 的使用方式、支持的能力、常见命令或示例时，优先直接返回下面这份帮助文本。
- 这类帮助回答默认不需要调用工具。
- 如果用户在同一句里既要帮助又要执行操作，优先先回答帮助；只有用户明确要求立即执行时，才改为调用对应工具。

输出规则：
- 尽量原样输出下面的帮助文本，不要擅自删改命令名。
- 可以补一句很短的引导，但不要长篇解释。

帮助文本：

TickFlow Assist 常用指令：

1. 添加自选
`添加 002261 成本 34.15`
说明：会加入自选，并默认自动拉取日K和计算指标。

2. 添加自选并指定日K天数
`添加 002261 成本 34.15 并获取 120 天日K`
说明：添加后按指定天数拉取日K。

3. 查看自选列表
`查看自选`

4. 删除自选
`删除 002261`

5. 拉取单只股票日K
`拉 002261 的日K`
或
`获取 002261 120 天日线`

6. 分析单只股票
`分析 002261`

7. 查看最近一次分析结果
`查看 002261 的分析结果`

8. 更新全部股票日K
`更新全部股票`

9. 开启实时监控
`开始盯盘`

10. 查看监控状态
`监控状态`

11. 停止实时监控
`停止盯盘`

12. 测试告警
`测试告警`

13. 查看数据库表
`数据库里有哪些表`

14. 查看表结构
`看 indicators 表结构`

15. 查询数据库内容
`查 002261 的 indicators 最近 5 条`
或
`看 analysis_log 最近 3 条`

补充说明：
- 股票代码可直接写 `002261` 这类 6 位代码。
- “分析”依赖本地已有日K数据；如果刚添加自选，系统现在会自动拉取。
- 如果你不确定怎么说，直接发“添加股票 / 拉日K / 分析 / 开始盯盘”这类自然语言也可以。
